I used Flexbox for the header and container elements to create a well-structured layout, ensure consistent spacing between elements, and control how content was distributed across the available space on the page. In the header, I used justify-content: space-between to position the title and menu icon at opposite ends of the header and align-items: center to vertically align them.
I used CSS Grid for the card gallery layout. Grid allowed me to arrange the cards into four columns on desktop screens, two columns on tablet screens, and one column on mobile screens. This made the layout responsive and easy to manage across different screen sizes.
Flexbox is generally better suited for one-dimensional layouts because it focuses on arranging items in either rows or columns. It provides simple and effective control over spacing, positioning, and alignment of content along a single direction. 
CSS Grid is generally better suited for multi-column layouts because it allows content to be organized into rows and columns simultaneously. This makes it ideal for creating structured and responsive layouts such as card galleries.
One challenge I encountered while implementing responsive design was ensuring that the correct layout appeared at different viewport sizes. I also had to troubleshoot conflicts between my earlier Flexbox based card layout and the final CSS Grid implementation, removing previous styling rules to ensure the cards displayed correctly across desktop, tablet, and mobile screens.
